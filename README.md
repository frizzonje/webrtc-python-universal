# Нативный клиент «Белой Берёзки» (macOS + Linux)

Браузер не умеет нормально захватывать **системный/игровой звук** при
демонстрации экрана. Этот клиент решает проблему: он подключается к тому же
mesh-WebRTC, что и ваши коллеги в браузере, и шлёт им:

- **микрофон** — вы говорите;
- **экран** — игра/рабочий стол, выбор «качество ⇄ фпс»;
- **звук системы** — то, что слышите вы: игра, музыка.

А голос собеседников проигрывает вам в наушники/колонки. Никаких изменений на
сервере и у браузерных коллег не требуется — протокол сигналинга, HMAC-пропуск
(`bb_pass`) и перфект-негоциация совпадают с веб-клиентом один-в-один.

Захват кросс-платформенный: **macOS** — через `avfoundation`, **Linux** — через
`x11grab` + PulseAudio. Входящий голос проигрывается через `sounddevice`
(PortAudio) на обеих системах.

---

## Быстрый старт (GUI)

```bash
cd native
./run.sh
```

Откроется простое окно в духе Discord: поля сервера/пароля/канала/имени,
выбор источников захвата и кнопка «Войти в канал». `run.sh` при первом запуске
сам поставит зависимости (на macOS — `ffmpeg` через Homebrew; на Ubuntu —
ffmpeg/pulseaudio-utils/python3-tk), создаст venv — дальше окно открывается
мгновенно.

---

## macOS — что нужно знать

1. **Разрешение «Запись экрана».** При первом «Войти в канал» macOS спросит
   доступ к записи экрана для приложения, из которого запущен клиент (обычно
   **Терминал** или **Python**). Включи его в *Системные настройки →
   Конфиденциальность и безопасность → Запись экрана* и **перезапусти клиент**.
   Без этого экран не передаётся (клиент сразу скажет об этом, а не зависнет).

2. **Микрофон.** Спросит доступ к микрофону — разреши. Это всё, что нужно для
   голоса и демонстрации картинки.

3. **Звук системы (опционально).** macOS не даёт захватывать звук системы
   напрямую — нужен виртуальный аудио-драйвер **BlackHole**:

   ```bash
   brew install blackhole-2ch
   ```

   Затем в *Audio MIDI Setup* создай **Multi-Output Device** из своих
   наушников/колонок + BlackHole 2ch и сделай его выводом системы. В клиенте
   поставь галочку «Транслировать звук системы» и выбери источником
   **BlackHole 2ch**. Кнопка **🔊 Звук системы** в окне покажет эту инструкцию.
   Без BlackHole всё остальное (микрофон + экран) работает как есть.

4. **Выбор источника** — выпадашка **«🖥 Что показывать»**: экраны и камеры
   перечисляются по именам, по умолчанию выбран «Capture screen 0». Захват идёт
   в нативном разрешении экрана; режим «Качество ⇄ Легче» меняет частоту кадров
   и битрейт (масштабирование на macOS недоступно).

### Ярлык в меню приложений (запуск в один клик)

```bash
./run.sh            # один раз в терминале — ставит зависимости
./install-desktop.sh
```

После этого «Белая Берёзка (нативка)» появится в списке программ Ubuntu —
можно закрепить на панель и запускать мышкой, без терминала.

### Что в окне

- **Поля** сервер/пароль/канал/имя (предзаполнены из `.env` проекта).
- **🖥 Что показывать** — выпадашка экранов/камер (что транслировать).
- **🎤 Микрофон** и **🎧 Слышать в** — выбор устройств ввода/вывода из списка.
- **«Транслировать звук системы»** — галка + выбор аудио-источника (macOS — BlackHole).
- **🔄 Обновить устройства** — перечитать список (подключил наушники — нажми).
- **Качество демонстрации** — сегмент `🎬 Качество` / `⚡ Легче`. В канале этот
  переключатель работает **на лету** — без переподключения.
- В канале: список «в эфире», кнопка **микрофона** (вкл/mute) и **Выйти**, журнал.
- Кнопка **🔊 Звук системы** (macOS) / **🎚 Анти-эхо** (Linux) — настройка
  трансляции звука системы.

> Сервер по умолчанию `https://192.168.0.138`, пароль `123` (из `.env` проекта).
> TLS у Caddy самоподписанный → проверка сертификата выключена по умолчанию.
> Можно префилить поля флагами: `./run.sh --server https://… --room myroom`.

## Консольный режим (без GUI)

```bash
./run.sh --cli --room general --name "Командир" --mode quality
# или напрямую:
./.venv/bin/python bb_native.py --room general --name "Командир"
```

---

## Режимы качества

| Режим          | Захват     | Под что                          |
|----------------|------------|----------------------------------|
| `--mode quality` | 1920×1080 @ 60 | детализация + плавность                |
| `--mode fps`     | 1280×720 @ 60  | плавность (динамичные игры)       |

Точная подстройка: `--width 2560 --height 1440 --fps 30`,
`--start-bitrate 6000 --max-bitrate 12000` (кбит/с).

## Звук: какой источник захватывать

Посмотреть, что есть в системе:

```bash
python bb_native.py --list-audio
```

- **Микрофон** — `--mic-source default` (или конкретный `alsa_input...`).
- **Системный звук** — `--system-source @DEFAULT_MONITOR@` (по умолчанию) или
  `--system-source <имя_sink>.monitor`.

### Чтобы коллеги не слышали сами себя (анти-эхо)

Если стримить `@DEFAULT_MONITOR@`, в захват попадёт и голос собеседников
(вы же выводите его в тот же приёмник) → они услышат эхо. Чистое решение —
отдельный виртуальный приёмник для игры:

```bash
python bb_native.py --setup-sink        # создаст sink 'bb_stream' + loopback
# в pavucontrol → Playback переведите ЗВУК ИГРЫ на «BB_Stream»
python bb_native.py --room general --system-source bb_stream.monitor
```

Тогда в трансляцию уходит только звук игры, а голос коллег играет в ваш обычный
приёмник и обратно не возвращается. Сам игру слышите через loopback.

Отключить системный звук вовсе: `--no-system-audio`. Микрофон: `--no-mic`.

## Если окно не открывается (ModuleNotFoundError: tkinter)

Значит `.venv` создан без доступа к системному tkinter. Пересоздайте:

```bash
rm -rf .venv && ./run.sh
```

## Wayland

`x11grab` работает в сессии **Xorg** (на экране входа GNOME — шестерёнка →
«Ubuntu on Xorg»). Под чистым Wayland захват экрана через этот путь недоступен;
самый простой обходной путь для показа — войти в сессию Xorg. (Игры под
Proton/Steam обычно и так в XWayland — но сам захват требует Xorg-сессии.)

## Как это работает (для своих)

- **Сигналинг** — `socket.io` к Caddy (`/socket.io`), события из `@bb/shared`:
  `join / peers / peer-joined / offer / answer / ice-candidate / peer-left`.
- **Пропуск** — кука `bb_pass = ${exp}.${base64url(HMAC-SHA256(exp,
  'bb-auth-v1:'+пароль))}`, формат совпадает с `packages/shared/src/auth.ts`.
- **Mesh без glare** — вы заходите в готовую комнату как newcomer и шлёте offer
  всем из списка `peers`; ваш offer содержит все 3 дорожки. Порядок дорожек
  (микрофон → видео → звук системы) совпадает с веб-клиентом, поэтому браузерный
  микшер верно раскладывает «голос» и «звук демонстрации». Поздним гостям
  отвечаем и до-догоняем экран отдельным offer'ом (ренеготиируем только мы →
  столкновений нет; rollback в aiortc не нужен).
- **Битрейт** — подсказки `x-google-*-bitrate` в SDP (как `boostVideoBitrate`),
  дальше aiortc подстраивается под обратную связь приёмника.

### Ограничения

- aiortc кодирует видео софтово (VP8) — для тяжёлых сцен на слабом CPU держите
  `--mode fps` или снизьте разрешение. Для максимума «картинки» — `quality`.
- Поздние гости (зашедшие уже во время трансляции) получают экран после
  до-offer'а — пара секунд задержки, это нормально.
